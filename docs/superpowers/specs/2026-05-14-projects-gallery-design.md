# ProjectsGallery — Design Spec
_Date: 2026-05-14_

## Overview

Replace the hardcoded `Projects.tsx` section with a new `ProjectsGallery.tsx` component that renders portfolio projects dynamically from the AWS Amplify Gen 2 backend. The component provides real-time updates via `observeQuery`, client-side search filtering, and a hidden seed button for deployment verification.

---

## Architecture & Data Flow

### Client Setup
- Use `generateClient<Schema>({ authMode: 'apiKey' })` from `aws-amplify/data`.
- The `Project` model uses `allow.publicApiKey()` authorization — the default `identityPool` mode will not work and must be overridden explicitly.
- Import `Schema` type from `../amplify/data/resource`.

### Real-Time Subscription
- A single `useEffect` on mount calls `client.models.Project.observeQuery().subscribe({ next: ({ items }) => setProjects(items) })`.
- Cleanup: `subscription.unsubscribe()` returned from `useEffect`.
- Projects are stored in a `projects` state array (`Schema['Project']['type'][]`).

### Search Filtering
- One `searchQuery` string state, updated on every keystroke.
- Derived `filtered` variable (not a separate state): filters `projects` where `title` or `techStack` includes the query, case-insensitive.
- No debounce needed — client-side filtering is instant.

### Tech Stack Tags
- `techStack` is stored as a comma-separated string in the schema (`a.string()`).
- Split on display: `project.techStack?.split(',').map(t => t.trim())`.

## Component Structure

**File:** `src/components/ProjectsGallery.tsx`

```
ProjectsGallery (default export)
├── Section header (sec-eyebrow + sec-title, id="work")
├── Search bar (glass-style input)
├── Project grid (1-col mobile / 3-col desktop)
│   └── ProjectCard (inline — no separate file needed)
│       ├── Title (h3)
│       ├── Description (p)
│       ├── Tech tags (glass-pill sm spans)
│       └── "View Live" link (glass-btn dark, conditional on liveUrl)
├── Empty state message (centered, when filtered list is empty)
└── Seed button (hidden at bottom)
```

---

## Layout & Styling

- **Section**: `id="work"` and className `project-section` — preserves Navbar anchor link without touching `Navbar.tsx`.
- **Header**: Matches existing site pattern with `sec-eyebrow` + `sec-title` class names.
- **Search bar**: `<input>` with `border border-white/10 bg-white/5 rounded-xl px-4 py-3 w-full max-w-md` (Tailwind v4). Placeholder: `"Search by title or tech stack…"`.
- **Grid**: `grid grid-cols-1 md:grid-cols-3 gap-6`.
- **Cards**: `rounded-2xl border border-white/10 bg-white/5 backdrop-blur-md p-6 flex flex-col gap-3` — glass-morphism matching the site aesthetic.
- **Tech tags**: Reuse existing `glass-pill sm` CSS class with `glass-icon` dot span inside, matching the hardcoded projects.
- **"View Live" link**: Reuse existing `glass-btn dark` CSS class. Only rendered when `liveUrl` is a non-empty string.
- **Empty state**: Centered `<p>` with muted styling when `filtered.length === 0`.

## Seed Button

- Hardcoded payload:
  ```ts
  { title: 'Test SaaS Project', description: 'A test project seeded to verify Amplify deployment.', techStack: 'React, TypeScript, AWS Amplify', liveUrl: '' }
  ```
- Calls `client.models.Project.create(payload)` on click.
- Positioned in `<div className="text-center mt-24">` at the bottom of the section.
- Styled `text-xs text-white/20 hover:text-white/60 transition-opacity cursor-pointer` — effectively invisible until hovered.

## Out of Scope

- Pagination or infinite scroll (not requested; client-side filter handles the full list).
- Image display (the `imageUrl` field exists in the schema but no image UI was requested).
- Authentication / admin gating for the seed button (it is intentionally temporary and obscured by styling).
- Animations / framer-motion on cards (the existing `Reveal` pattern is not carried over to this data-driven grid).
